## 2. Current architecture (one-paragraph truth)

xiaoguai ships as **one Rust binary (`xiaoguai-core`) over embedded SQLite**,
on **port 7600**, serving chat-ui at `/` and admin-ui at `/admin/`. **Single
implicit owner**; optional **HTTP Basic** gate (`auth.username`/`auth.password`,
empty = open localhost). **No** Postgres, Redis/Valkey, OIDC/JWT, Casbin/RBAC,
scopes, multi-tenancy, or Kubernetes/Helm. Cache is a process-local `DashMap`.
Delivery: docker-compose, native `.deb`/`.rpm`, bare-metal tarball, pip wheel
(all bundle the web UI). Subsystems: HMAC-chained audit + compliance export,
HotL suspend/resume governance, scheduler (cron/filewatch/webhook/proactive),
MCP two-way, RAG + citations, SQLite memory (BLOB f32 + cosine), agent-authored
skills (whitelist), eval framework, Planner/Worker/Critic orchestrator, 7 IM
adapters, L1/L2/L3 sandboxed code exec, opt-in observability.

  are request/response only.
- **Single binary + SQLite + zero external deps** AND governed â€” Hermes is
  light but ungoverned; enterprise tools are governed but heavy. xiaoguai is
  uniquely both.

  **persistent-workspace mode** + file/git tools reusing that L1 + HotL + audit
