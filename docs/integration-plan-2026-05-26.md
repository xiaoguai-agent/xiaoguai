## Wave 4 Python Keep-Both Regex Script

Save as `scripts/convoy-keep-both.py` and run during Wave 4 conflict resolution:

```python
#!/usr/bin/env python3
"""
Keep-both helper for AppState convoy merges.
Usage: python3 scripts/convoy-keep-both.py <file>
Resolves <<<<<<< / ======= / >>>>>>> conflict markers by keeping both sides.
"""
import re
import sys

def resolve_keep_both(path: str) -> None:
    with open(path, "r") as f:
        text = f.read()

    # Match conflict blocks and keep both sides
    pattern = re.compile(
        r"<<<<<<< .*?\n(.*?)=======\n(.*?)>>>>>>> .*?\n",
        re.DOTALL
    )

    def keep_both(m: re.Match) -> str:
        ours = m.group(1)
        theirs = m.group(2)
        # For AppState struct fields and router chains: keep both sides
        # Remove duplicate blank lines
        combined = ours.rstrip("\n") + "\n" + theirs
        return combined

    resolved = pattern.sub(keep_both, text)

    if resolved == text:
        print(f"No conflict markers found in {path}")
        return

    with open(path, "w") as f:
        f.write(resolved)
    print(f"Resolved conflict markers in {path} (keep-both)")


if __name__ == "__main__":
    for path in sys.argv[1:]:
        resolve_keep_both(path)
```

**Usage during convoy**:
```bash
# After git merge raises conflict on main.rs:
python3 scripts/convoy-keep-both.py crates/xiaoguai-core/src/main.rs
python3 scripts/convoy-keep-both.py crates/xiaoguai-api/src/state.rs
python3 scripts/convoy-keep-both.py crates/xiaoguai-cli/src/commands/mod.rs
# Verify, then:
cargo check -p xiaoguai-core -p xiaoguai-api
git add <resolved files>
git merge --continue
```

**Do NOT use keep-both on**:
- `Cargo.lock` (regenerate instead)
- `catalog/skill_packs.json` (manual JSON append)
- `docs/book/src/SUMMARY.md` (manual line append)
- Migration SQL files (rename 0016_personas → 0018_personas)
