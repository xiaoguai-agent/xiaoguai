## 6. Working conventions to keep

- Commit per logical task with `feat/fix/test/docs(vX.Y.Z): ...`.
- Every milestone closes with a doc update + `cargo test --workspace`
  number recorded + `clippy -D warnings` + `fmt --check` clean.
- Each milestone tagged + pushed: `git tag vX.Y && git push origin
  main && git push origin vX.Y`.
- Always provide a deferral list in the milestone plan doc вЂ” leave the
  tree shippable at every tag.
- Tests live next to the code they cover (`tests/` for crate
  integration; `#[cfg(test)] mod tests` for unit). Use scripted
  `MockBackend` + in-memory repos in `tests/common/`.
- Frontend changes: keep `@xiaoguai/shared` as the single source of
  truth for wire types вЂ” both chat-ui and admin-ui import from it.

## 7. Known sharp edges

- `xiaoguai-core serve` falls back to **MockBackend** when the
  `llm_providers` table is empty. Use `xiaoguai provider register` to
  populate it before running real traffic.
- `auth: None` and `authz: None` are the defaults. Set
  `XIAOGUAI_AUTH__REQUIRED=true` + provide JWKS / issuer / audience to
  flip both JWT verification and Casbin per-route enforcement on.
- The LlmBackend impl on LlmRouter uses `ResolveCtx::default()`, so the
  agent loop only sees the system-default + fallback chain вЂ” per-tenant
  routing requires a separate slice (deferred to v0.6.3).
- Feishu integration's signature algorithm matches the *common* spec
  variant; some Feishu apps use a slightly different scheme depending on
  encryption settings. Real-world testing pending.
- The 51 ignored tests are mostly testcontainer-based PG e2e вЂ” they pass
  locally if you have Docker but are not part of CI's fast path.
