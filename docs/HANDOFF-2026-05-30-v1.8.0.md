Sub-agent deviations worth knowing (full rationale in each PR body):
- **`WatchRunner` trait NOT changed** — Phase A introduced `WatcherIntrospector` in `xiaoguai-api/src/watchers.rs` (mirroring `TodayReader`/`UsageReader`) instead. Trait is the swap point for a future session-aware adapter.
- **Persona DTO** — real fields are `default_model`/`tool_allowlist`/`escalation_tier`, no `tags`/`memory_view_scope`. Role chips inferred from `system_prompt` until a `tags` column lands.
- **Skill Proposals approve** — backend has no `comment` field on `ApproveRequest`; frontend hard-codes `decided_by: "admin-ui"` sentinel until auth identity lands.
- **`<RequireScope>` fails open** when `/me/scopes` is unavailable — UX hint only, backend Casbin enforces (PHILO §17 anti-pattern: never trust a frontend gate for security).

## Sprint-11 backlog — 3 LLD-vs-impl drifts

Surfaced by S10b-8 Playwright e2e (each has a `test.fixme()` placeholder with embedded gap-close checklist in the spec header):

### Drift #1 — Audit pane lacks ChainBadge + Export UI

- **Spec**: LLD-ADMIN-UI-001 §4.2 (HMAC chain visualisation + compliance export)
- **Reality**: `frontend/admin-ui/src/panes/Audit.tsx` renders id/ts/actor/action/resource/hmac columns only — no green/amber/red ChainBadge column, no Export modal
- **e2e fixme** in `frontend/e2e/tests/admin-ui/admin-audit-export.spec.ts` (2 cases with 4-step gap-close checklist)
- **Backend**: `POST /v1/audit/exports` is mounted (sprint-7 T5 / PR #74); just needs UI

### Drift #2 — Chat SSE reconnect banner unimplemented

- **Spec**: LLD-CHAT-UI-001 §4.7 (failure mode: SSE drops → banner appears → exponential backoff reconnect)
- **Reality**: `XiaoguaiClient::sendMessage()` calls `onError` on stream failure but doesn't retry or render anything. Partial-text preservation already works (covered by passing test).
- **e2e fixme** in `frontend/e2e/tests/chat-ui/chat-sse-reconnect.spec.ts` (1 case)

### Drift #3 — HotL inline approve/reject deferred

- **Spec**: LLD-CHAT-UI-001 §4.3.1 (HotL pending → inline Approve/Deny/Approve-and-remember buttons in the conversation)
- **Reality**: `HotlBanner` is link-out to admin only — no inline buttons
- **e2e fixme** in `frontend/e2e/tests/chat-ui/chat-hotl-suspend-resume.spec.ts` (1 case)

All three are real gaps the LLDs already specify. Sprint-11 Step 1 is likely a small refresh to the LLDs marking the gaps as in-flight, then Step 2 task plan, then implementation.
