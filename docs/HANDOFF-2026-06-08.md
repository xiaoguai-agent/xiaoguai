## 3. CI: the runner-death war (6 rounds, partially won)

Symptom: "runner lost communication" at hang+45 min, killing the gate for
days. Full forensics log: **issue #243** + memory `ci-gotchas`.

- **KILLED: the per-crate rustc rebuild hang.** `cargo test -p` re-resolves
  features vs `--workspace` → silent recompiles; one such rustc spun forever.
  Fix: **cargo-nextest archive** — build once (workspace unification), per-
  crate steps only run (2-3 s each). Doctests get their own step.
- **QUARANTINED, NOT root-caused: a test-execution hang in
  `xiaoguai-mcp-exec`.** Defeats every in-job containment (cgroup jail,
  swap=0, nested timeouts, file-redirect — even kill-the-waiter never
  returned control). Now in its own `continue-on-error` job: poll-don't-wait
  execution + git-push beacon (`refs/heads/ci-beacon-<run_id>`) collects
  evidence on every recurrence. Local clue: nextest flags
  `exec::tests::stderr_is_redacted_when_configured` as LEAKY ~1/7 runs.
- Gate shape now: build-test (35 crates via archive + doctests + eval) +
  quarantined mcp-exec + migrations smoke. All green on main `2d70268`.
