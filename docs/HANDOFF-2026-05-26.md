## 8. Lessons captured

- **Agents writing test vectors transcribed wrong values twice** —
  Slack signature docs vector and AWS SigV4 docs vector both failed
  to round-trip against independent reference implementations
  (Python). Always recompute when testing crypto.
- **`cargo update --workspace` is the right way to resolve Cargo.lock
  conflicts** — `git checkout --theirs` left embedded conflict markers
  when the file was already staged; `rm Cargo.lock && cargo generate-lockfile`
  is the reliable path.
- **AppState convoy** — the python "keep both" regex resolver
  (`pat.sub(lambda m: m.group(1) + m.group(2), s)`) handled all
  HIGH-risk merges in seconds; manually patch only the new test
  file each branch introduces (it doesn't yet know about the other
  branches' fields).
- **Test/source skew between branches** — when a wave-3 branch's
  test file references APIs that don't exist on main (the agent
  invented `EchoBackend`, `InMemoryMessageRepo::new()`, etc.), use
  the existing test file (e.g. `tests/api.rs`) as the template and
  rewrite the new test file's helpers to match.
- **Don't bump `rust-version` proactively** — `aws-smithy-types` 1.91
  forced it after the audit-s3 merge; pin the dep instead unless CI
  toolchain is already on the new version.
