# Xiaoguai v0.5 Inner Loop Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make the agent loop actually intelligent: real Postgres + Valkey persistence, OIDC auth + RBAC, multi-backend LLM router, MCP client with all three transports + per-tenant supervisor pool, full ReAct loop with parallel tool dispatch, and a streaming HTTP/WS API server. CLI gains a `--server` mode that talks to a remote `xiaoguai-core`.

**Architecture:** Builds on v0.1 scaffold. Each sub-milestone (v0.5.1 → v0.5.6) lands as a tagged release with green CI. Sub-milestones are sequential — each builds on the previous.

**Tech Stack:** Rust 1.88+, sqlx (Postgres), redis (Valkey), tokio, axum, tonic, casbin, jsonwebtoken, MCP Rust SDK (modelcontextprotocol/rust-sdk), seccompiler, nix (cgroup v2), tower middleware.

```sql
CREATE EXTENSION IF NOT EXISTS pgcrypto;

CREATE TABLE tenants (
    id              TEXT PRIMARY KEY,
    name            TEXT NOT NULL UNIQUE,
    display_name    TEXT NOT NULL,
    status          TEXT NOT NULL DEFAULT 'active',
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE users (
    id              TEXT PRIMARY KEY,
    tenant_id       TEXT NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
    email           TEXT NOT NULL,
    display_name    TEXT NOT NULL,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_login_at   TIMESTAMPTZ,
    UNIQUE (tenant_id, email)
);

CREATE TABLE user_roles (
    user_id         TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role            TEXT NOT NULL,
    PRIMARY KEY (user_id, role)
);

CREATE TABLE sessions (
    id              TEXT PRIMARY KEY,
    tenant_id       TEXT NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
    user_id         TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    title           TEXT,
    model           TEXT NOT NULL,
    status          TEXT NOT NULL DEFAULT 'active',
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX ix_sessions_user_updated ON sessions (user_id, updated_at DESC);

CREATE TABLE messages (
    id              TEXT PRIMARY KEY,
    session_id      TEXT NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    role            TEXT NOT NULL,
    content         JSONB NOT NULL,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX ix_messages_session ON messages (session_id, created_at);

-- RLS: all tenant-scoped tables.
ALTER TABLE tenants ENABLE ROW LEVEL SECURITY;
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE sessions ENABLE ROW LEVEL SECURITY;
ALTER TABLE messages ENABLE ROW LEVEL SECURITY;

CREATE POLICY tenant_isolation ON users
    USING (tenant_id = current_setting('app.current_tenant_id', true));
CREATE POLICY tenant_isolation ON sessions
    USING (tenant_id = current_setting('app.current_tenant_id', true));
CREATE POLICY tenant_isolation_messages ON messages
    USING (session_id IN (SELECT id FROM sessions));
```

```rust
pub async fn connect(url: &str, max_conn: u32) -> anyhow::Result<PgPool> {
```

## v0.5.4 — Full ReAct agent loop — **DONE 2026-05-22 (tag `v0.5.4`)**

**Goal:** Replace the v0.1 single-turn `run_once` with a real ReAct loop: think → tool_call(s) parallel → observe → loop until done.

**Scope-cut shipped in v0.5.4:**
- OpenAI-style tool calls plumbed through `ChatRequest` / `Message` / `ChatChunk` /
  `OpenAiCompatBackend` (streamed tool-call delta accumulation) + scripted
  `MockBackend::with_script` for multi-turn tests.
- `xiaoguai-agent::Toolbox` (flat `tool_name → (Arc<dyn McpClient>, ToolDescriptor)`
  with duplicate rejection) + `to_specs()` → `ChatRequest.tools`.
- `ReactAgent::run_stream` / `run_to_completion`: think → parallel
  `futures::join_all` tool dispatch → observe → loop. Cancellation polled
  between iterations and before each fanout; max-iteration guard caps runaway
  loops. Tool errors injected as structured `{error, text}` JSON in the
  `Role::Tool` message so the model can recover.
- `history::slide` sliding-window trim (system msgs preserved, dangling
  `Role::Tool` heads stripped to avoid provider 400s).
- `AgentEvent` enum (`TextDelta` / `ToolCallStarted` / `ToolCallFinished` /
  `IterationCompleted` / `Done` / `Error`) streamed via
  `tokio::sync::mpsc` + `ReceiverStream`; agent tolerates dropped receivers.

**Deferred to v0.5.4.1:**
- LLM-driven history summary + manual `/compact` slash command (sliding window
  alone is enough to keep loops bounded).
- `/plan` mode (T6 from the original outline).
- Streaming tool_call deltas surfaced to caller (currently only the terminal
  chunk carries `tool_calls`).
- Ollama native tool-calling support (currently `Role::Tool` renders as `user`).

### Tasks (outline)

- **T1:** Tool call dispatcher — emit MCP tool list to LLM as OpenAI tool schema, parse tool_calls back from LLM stream. **DONE**
- **T2:** Parallel tool execution via `futures::join_all`. **DONE**
- **T3:** Conversation history compression — sliding window. (LLM summary + `/compact` deferred to v0.5.4.1.) **DONE**
- **T4:** Max-iteration guard + `CancellationToken` + cooperative shutdown. **DONE**
- **T5:** Tool result formatting + injection into next LLM call. **DONE**
- **T6:** Optional `/plan` mode — explicit plan tree + step execution. **DEFERRED → v0.5.4.1**
- **T7:** Stream events to caller (text deltas, tool_call started/finished, errors). **DONE**
- **T8:** Integration tests with MockBackend producing scripted tool_calls. **DONE (9 e2e tests + 6 unit tests)**
- **T9:** v0.5.4 tag. **DONE**

## v0.5.5 — xiaoguai-api HTTP server — **DONE 2026-05-22 (tag `v0.5.5`)**

**Goal:** Expose REST + SSE endpoints. Frontend can talk to this. CLI remote mode in v0.5.6 talks to this.

**Scope-cut shipped in v0.5.5:**
- `xiaoguai-api` upgraded from skeleton to a real axum router with:
  - `AppState` (session + message repos via trait objects, `Arc<dyn LlmBackend>`,
    shared `Toolbox`, agent defaults, `CancelRegistry` for per-session
    cooperative cancellation).
  - Middleware: `TraceLayer` + permissive `CorsLayer` (origin tightening
    moves to v0.5.5.1 with auth).
  - Endpoints: `GET /healthz`, `POST /v1/sessions`, `GET /v1/sessions/:id`,
    `GET /v1/sessions/:id/messages`, `POST /v1/sessions/:id/messages` (SSE),
    `POST /v1/sessions/:id/cancel`.
- `POST .../messages` drives `ReactAgent::run_stream`, encodes each
  `AgentEvent` as a typed SSE event (`text_delta` / `tool_call_started` /
  `tool_call_finished` / `iteration_completed` / `done` / `error`), and
  spawns a separate finalize task that persists newly-appended messages
  after the join handle resolves. Receiver drop in the SSE stream does not
  block persistence.
- `convert::llm_to_domain` / `convert::domain_to_llm` bridge between the
  agent's `xiaoguai_llm::Message` and the persistence-layer
  `xiaoguai_types::Message` (block-list with IDs + timestamps); the
  `is_error` flag is recovered from the `{error, text}` payload the agent
  injects for failed tool calls.
- `ApiError` → JSON `{code, message}` body with stable error slugs.
- In-memory `SessionRepository` + `MessageRepository` impls live in
  `tests/common` and back the integration suite (real PG repos already
  exist from v0.5.1; testcontainer-backed e2e is its own ignored suite).

**Deferred to v0.5.5.1:**
- Auth middleware (OIDC JWT validation; `xiaoguai-auth` crate exists but
  isn't wired into request extraction yet).
- RBAC enforcement via Casbin.
- Per-request RLS context (`SET LOCAL app.current_tenant_id`).
- WebSocket fallback (`/v1/sessions/:id/ws`).
- OpenAPI spec generation via `utoipa`.
- `GET /v1/mcp/servers` + `POST /v1/admin/tenants` + `GET /v1/admin/audit`.
- `xiaoguai-core` wiring `Serve` to actually bind the router.

### Tasks (outline)

- **T2:** Endpoints (5 of 8 from the original outline — `/v1/sessions` + messages + cancel; `/v1/mcp/servers` and `/v1/admin/*` deferred). **DONE**

## Risk & Fallback

| Risk | Mitigation |
|---|---|
| testcontainers slow on dev machines (Postgres image pull) | Use `:alpine` variant + cache image in CI |
| MCP Rust SDK API still in flux | Pin to specific version; fork if blocking issues |
| seccomp/cgroup requires Linux | Document macOS dev limitation; CI runs on Linux |
| RLS adds complexity to every query | Helper macro `with_tenant!(pool, tenant_id, query)` that sets the session var |
| Casbin policy file diverges from code | Store policies in PG + reload on change |
| JWT cache invalidation | TTL-based + max-age; document operator runbook |
