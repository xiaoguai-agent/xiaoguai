# Xiaoguai v0.5 Inner Loop Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make the agent loop actually intelligent: real Postgres + Valkey persistence, OIDC auth + RBAC, multi-backend LLM router, MCP client with all three transports + per-tenant supervisor pool, full ReAct loop with parallel tool dispatch, and a streaming HTTP/WS API server. CLI gains a `--server` mode that talks to a remote `xiaoguai-core`.

**Architecture:** Builds on v0.1 scaffold. Each sub-milestone (v0.5.1 → v0.5.6) lands as a tagged release with green CI. Sub-milestones are sequential — each builds on the previous.

**Tech Stack:** Rust 1.88+, sqlx (Postgres), redis (Valkey), tokio, axum, tonic, casbin, jsonwebtoken, MCP Rust SDK (modelcontextprotocol/rust-sdk), seccompiler, nix (cgroup v2), tower middleware.

`crates/xiaoguai-storage/migrations/0001_initial.sql`:
```sql
CREATE EXTENSION IF NOT EXISTS pgcrypto;

CREATE TABLE tenants (
    id              TEXT PRIMARY KEY,
    name            TEXT NOT NULL UNIQUE,
    display_name    TEXT NOT NULL,
    status          TEXT NOT NULL DEFAULT 'active',
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE users (
    id              TEXT PRIMARY KEY,
    tenant_id       TEXT NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
    email           TEXT NOT NULL,
    display_name    TEXT NOT NULL,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_login_at   TIMESTAMPTZ,
    UNIQUE (tenant_id, email)
);

CREATE TABLE user_roles (
    user_id         TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role            TEXT NOT NULL,
    PRIMARY KEY (user_id, role)
);

CREATE TABLE sessions (
    id              TEXT PRIMARY KEY,
    tenant_id       TEXT NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
    user_id         TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    title           TEXT,
    model           TEXT NOT NULL,
    status          TEXT NOT NULL DEFAULT 'active',
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX ix_sessions_user_updated ON sessions (user_id, updated_at DESC);

CREATE TABLE messages (
    id              TEXT PRIMARY KEY,
    session_id      TEXT NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    role            TEXT NOT NULL,
    content         JSONB NOT NULL,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX ix_messages_session ON messages (session_id, created_at);

-- RLS: all tenant-scoped tables.
ALTER TABLE tenants ENABLE ROW LEVEL SECURITY;
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE sessions ENABLE ROW LEVEL SECURITY;
ALTER TABLE messages ENABLE ROW LEVEL SECURITY;

CREATE POLICY tenant_isolation ON users
    USING (tenant_id = current_setting('app.current_tenant_id', true));
CREATE POLICY tenant_isolation ON sessions
    USING (tenant_id = current_setting('app.current_tenant_id', true));
CREATE POLICY tenant_isolation_messages ON messages
    USING (session_id IN (SELECT id FROM sessions));
```

**Step 2: PG pool wrapper**

```rust
use sqlx::postgres::{PgPool, PgPoolOptions};

pub async fn connect(url: &str, max_conn: u32) -> anyhow::Result<PgPool> {
    Ok(PgPoolOptions::new()
        .max_connections(max_conn)
        .connect(url)
        .await?)
}

pub async fn migrate(pool: &PgPool) -> anyhow::Result<()> {
    sqlx::migrate!("./migrations").run(pool).await?;
    Ok(())
}
```

## v0.5.4 — Full ReAct agent loop

**Goal:** Replace the v0.1 single-turn `run_once` with a real ReAct loop: think → tool_call(s) parallel → observe → loop until done.

### Tasks (outline)

- **T1:** Tool call dispatcher — emit MCP tool list to LLM as OpenAI tool schema, parse tool_calls back from LLM stream.
- **T2:** Parallel tool execution via `tokio::join_all`.
- **T3:** Conversation history compression — sliding window + LLM-driven summary at threshold + manual `/compact`.
- **T4:** Max-iteration guard + cancellation token + cooperative shutdown.
- **T5:** Tool result formatting + injection into next LLM call.
- **T6:** Optional `/plan` mode — explicit plan tree + step execution.
- **T7:** Stream events to caller (text deltas, tool_call started/finished, errors).
- **T8:** Integration tests with MockBackend producing scripted tool_calls.
- **T9:** v0.5.4 tag.

## Risk & Fallback

| Risk | Mitigation |
|---|---|
| testcontainers slow on dev machines (Postgres image pull) | Use `:alpine` variant + cache image in CI |
| MCP Rust SDK API still in flux | Pin to specific version; fork if blocking issues |
| seccomp/cgroup requires Linux | Document macOS dev limitation; CI runs on Linux |
| RLS adds complexity to every query | Helper macro `with_tenant!(pool, tenant_id, query)` that sets the session var |
| Casbin policy file diverges from code | Store policies in PG + reload on change |
| JWT cache invalidation | TTL-based + max-age; document operator runbook |
