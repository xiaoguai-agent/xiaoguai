# Xiaoguai v0.5.3 — MCP client + supervisor (scope-cut) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Ship a working MCP **stdio** client + persistent server registry + minimal per-tenant supervisor + admin CLI, validated against the real `@modelcontextprotocol/server-filesystem` reference impl. This unlocks v0.5.4 (ReAct loop) which needs `list_tools` + `call_tool` to exist.

**Architecture:**

- `xiaoguai-mcp` becomes a real crate, not a skeleton. Re-exports types from `rmcp 1.7` and provides our own thin `McpClient` trait so the supervisor and agent layers don't see the SDK surface directly.
- `McpSupervisor` owns a `HashMap<McpKey, McpHandle>` (key = `(tenant, server_id, version)`). v0.5.3 ships the minimum: `start / get / stop / list_active`. LRU eviction + idle-timeout + crash-restart land in **v0.5.3.1**.
- Server **registry** lives in PG (`mcp_servers` table, RLS scoped) and is administered via `xiaoguai mcp register/list/remove`, mirroring the v0.5.2 `provider` subcommand pattern exactly.
- **Out of scope (deferred to v0.5.3.1 or later):**
  - SSE and streamable-HTTP transports (rmcp supports them; we just don't wire yet)
  - cgroup v2 + seccomp + netns sandboxing (Linux-only; agent dev is macOS — defer to a Linux CI lane)
  - Ping-every-30s health checks + Prometheus metrics
  - Default-deny network policy + per-MCP allowlist

**Tech Stack:** `rmcp 1.7` (features: `client`, `transport-child-process`, `transport-async-rw`, `macros`), `tokio::process::Command`, `sqlx` (Postgres), `parking_lot::Mutex`, `assert_cmd`.

**Pre-existing state (tag `v0.5.2`):**

- `crates/xiaoguai-mcp/` exists as a 3-line skeleton (`lib.rs` doc only).
- `crates/xiaoguai-storage/migrations/` has 0001–0004 applied.
- `crates/xiaoguai-cli/` has `chat` + `provider` subcommands and a working `commands/` module pattern.
- `crates/xiaoguai-types/` has `McpServerInstanceId` newtype but **no** `McpServer` struct yet.

## Design notes

### `McpClient` trait shape (T1)

```rust
#[async_trait]
pub trait McpClient: Send + Sync {
    /// Initialize handshake and cache server capabilities.
    async fn initialize(&self) -> Result<ServerInfo, McpError>;

    /// List all tools exposed by the server.
    async fn list_tools(&self) -> Result<Vec<ToolDescriptor>, McpError>;

    /// Invoke a tool by name. Arguments are JSON; result is a typed envelope.
    async fn call_tool(&self, name: &str, args: JsonValue) -> Result<ToolResult, McpError>;

    /// Best-effort shutdown of the underlying transport.
    async fn shutdown(&self) -> Result<(), McpError>;
}
```

`ToolDescriptor` and `ToolResult` are **our** types (re-shaped from rmcp's), so v0.5.4's agent loop doesn't import rmcp. Two reasons:

1. rmcp's surface changes between versions; we own the boundary.
2. Some rmcp fields (e.g. `_meta`) are noise for the agent.

### Why not just re-export rmcp?

We tried that pattern with `pyVmomi`/`avisdk` in a sibling project (see `myskills/CLAUDE.md` 踩坑 #32) — re-exporting SDK types leaks lifetime/serde constraints into every downstream crate and breaks the next SDK bump. The thin-wrapper cost (~80 LoC) is one-time, and we already pay it for `LlmBackend` against `async-openai`.

### Supervisor minimalism (T3)

For v0.5.3 the supervisor is essentially a typed `HashMap`:

```rust
pub struct McpSupervisor {
    clients: parking_lot::Mutex<HashMap<McpKey, Arc<dyn McpClient>>>,
}
```

What we **don't** ship in v0.5.3:

- Idle timeout (clients live forever until `stop`)
- Crash restart (caller gets an error if the process died; restart is manual)
- LRU eviction (memory bounded by registry size, fine for v0.5.3 where MCP count per tenant is ≤10)
- Metrics

These all land in **v0.5.3.1** which is dedicated to operational hardening. The acceptance test in C6 only proves the lifecycle skeleton; v0.5.3.1 will add chaos tests.

### Why we mock-test in-process for C3 instead of always invoking the FS server

Spawning `npx` makes the test slow (~3s warm), flaky (network for first install), and brittle (different node versions). The in-process scripted server (T1 Step 2) gives us deterministic, fast (≤50ms) coverage of the protocol surface. C7's e2e test stays `#[ignore]` for the same reason the storage repo tests do.

### macOS dev / Linux CI matrix

v0.5.3 has no platform-specific code. The Linux-only sandboxing in v0.5.3.1 will gate `xiaoguai-mcp::sandbox` behind `#[cfg(target_os = "linux")]` with a no-op fallback that logs "running unsandboxed (dev only)". CI will run the unsandboxed lane on macOS-latest and the sandboxed lane on ubuntu-latest.

```rust
//! layer in v0.5.4 only ever sees these — not rmcp's surface.

use serde::{Deserialize, Serialize};

    pub version: String,
}

#[derive(Debug, Clone, Serialize, Deserialize)]

    pub name: String,
    pub description: Option<String>,

    pub input_schema: JsonValue,
}

#[derive(Debug, Clone, Serialize, Deserialize)]

    /// concatenate text blocks here and surface non-text blocks in `blocks`.
    pub text: String,
    pub blocks: Vec<ContentBlock>,
    pub is_error: bool,
}

}
```

```rust
//! Minimal MCP server that speaks the protocol over stdio just enough for
//! our integration test. We hand-roll the JSON-RPC layer rather than depend
//! on rmcp's `server` feature to keep this fixture's compile time low.

use std::io::{self, BufRead, Write};

fn main() {
    let stdin = io::stdin();
    let stdout = io::stdout();
    let mut out = stdout.lock();
    for line in stdin.lock().lines().flatten() {
        let req: serde_json::Value = match serde_json::from_str(&line) {
            Ok(v) => v,
            Err(_) => continue,
        };
        let id = req.get("id").cloned().unwrap_or(serde_json::Value::Null);
        let method = req.get("method").and_then(|m| m.as_str()).unwrap_or("");
        let resp = match method {
            "initialize" => serde_json::json!({
                "jsonrpc": "2.0", "id": id,
                "result": {
                    "protocolVersion": "2025-06-18",
                    "capabilities": { "tools": {} },
                    "serverInfo": { "name": "mock", "version": "0.1.0" }
                }
            }),
            "tools/list" => serde_json::json!({
                "jsonrpc": "2.0", "id": id,
                "result": {
                    "tools": [
                        { "name": "echo", "description": "echoes its input",
                          "inputSchema": { "type": "object",
                            "properties": { "msg": { "type": "string" } },
                            "required": ["msg"] } }
                    ]
                }
            }),
            "tools/call" => {
                let args = req.get("params").and_then(|p| p.get("arguments")).cloned()
                    .unwrap_or(serde_json::Value::Null);
                let msg = args.get("msg").and_then(|m| m.as_str()).unwrap_or("");
                serde_json::json!({
                    "jsonrpc": "2.0", "id": id,
                    "result": {
                        "content": [{ "type": "text", "text": format!("echo: {msg}") }],
                        "isError": false
                    }
                })
            }
            "notifications/initialized" => continue, // notification, no response
            _ => serde_json::json!({
                "jsonrpc": "2.0", "id": id,
                "error": { "code": -32601, "message": format!("method not found: {method}") }
            }),
        };
        writeln!(out, "{resp}").ok();
        out.flush().ok();
    }
}
```

**Step 5: Implement `StdioMcpClient` on top of rmcp**

`crates/xiaoguai-mcp/src/stdio.rs`:

```rust
//! Stdio-transport MCP client built on rmcp's `TokioChildProcess` transport.

use std::ffi::OsStr;
use std::path::Path;

use async_trait::async_trait;
use rmcp::model::{CallToolRequestParam, ClientCapabilities, ClientInfo, Implementation};
use rmcp::service::RunningService;
use rmcp::transport::TokioChildProcess;
use rmcp::{RoleClient, ServiceExt};
use serde_json::Value as JsonValue;
use tokio::process::Command;

use crate::client::McpClient;
use crate::error::{McpError, McpResult};
use crate::types::{ContentBlock, ServerInfo, ToolDescriptor, ToolResult};

pub struct StdioMcpClient {
    service: RunningService<RoleClient, ()>,
}

impl StdioMcpClient {
    /// Spawn the given binary and complete the MCP `initialize` handshake.
    pub async fn spawn<P, S>(
        program: P,
        args: &[&str],
        envs: &[(S, S)],
    ) -> McpResult<Self>
    where
        P: AsRef<Path>,
        S: AsRef<OsStr>,
    {
        let mut cmd = Command::new(program.as_ref());
        cmd.args(args);
        for (k, v) in envs {
            cmd.env(k.as_ref(), v.as_ref());
        }
        let transport = TokioChildProcess::new(cmd)
            .map_err(|e| McpError::Transport(format!("spawn: {e}")))?;
        let client_info = ClientInfo {
            protocol_version: Default::default(),
            capabilities: ClientCapabilities::default(),
            client_info: Implementation {
                name: "xiaoguai".into(),
                version: env!("CARGO_PKG_VERSION").into(),
            },
        };
        let service = client_info
            .serve(transport)
            .await
            .map_err(|e| McpError::Protocol(format!("initialize: {e}")))?;
        Ok(Self { service })
    }
}

#[async_trait]
impl McpClient for StdioMcpClient {
    async fn initialize(&self) -> McpResult<ServerInfo> {
        let info = self.service.peer_info();
        Ok(ServerInfo {
            name: info.server_info.name.clone(),
            version: info.server_info.version.clone(),
        })
    }

    async fn list_tools(&self) -> McpResult<Vec<ToolDescriptor>> {
        let resp = self
            .service
            .list_tools(Default::default())
            .await
            .map_err(|e| McpError::Protocol(format!("list_tools: {e}")))?;
        Ok(resp.tools.into_iter().map(|t| ToolDescriptor {
            name: t.name.into_owned(),
            description: t.description.map(|d| d.into_owned()),
            input_schema: serde_json::to_value(&t.input_schema).unwrap_or(JsonValue::Null),
        }).collect())
    }

    async fn call_tool(&self, name: &str, args: JsonValue) -> McpResult<ToolResult> {
        let arguments = args.as_object().cloned();
        let resp = self
            .service
            .call_tool(CallToolRequestParam {
                name: name.to_string().into(),
                arguments,
            })
            .await
            .map_err(|e| McpError::Protocol(format!("call_tool: {e}")))?;

        let is_error = resp.is_error.unwrap_or(false);
        let mut text = String::new();
        let mut blocks: Vec<ContentBlock> = Vec::new();
        for c in resp.content {
            match c.raw {
                rmcp::model::RawContent::Text(t) => {
                    if !text.is_empty() {
                        text.push('\n');
                    }
                    text.push_str(&t.text);
                    blocks.push(ContentBlock::Text { text: t.text });
                }
                rmcp::model::RawContent::Image(i) => blocks.push(ContentBlock::Image {
                    mime_type: i.mime_type,
                    data_base64: i.data,
                }),
                rmcp::model::RawContent::Resource(r) => blocks.push(ContentBlock::Resource {
                    uri: r.resource.uri,
                }),
                _ => {} // ignore unknown block types
            }
        }
        Ok(ToolResult { text, blocks, is_error })
    }

    async fn shutdown(&self) -> McpResult<()> {
        // RunningService cancels its task on drop; explicit cancel would consume
        // self. Best-effort: do nothing here, rely on Drop.
        Ok(())
    }
}
```

```rust
//! Minimal MCP supervisor — owns active client instances keyed by
//! (tenant, server_name, version).
//!
//! v0.5.3 ships only start/get/stop/list_active. Idle-timeout, LRU
//! eviction, crash-restart, and health-check pings land in v0.5.3.1.

use std::collections::HashMap;
use std::sync::Arc;

use parking_lot::Mutex;

use crate::client::McpClient;
use crate::error::{McpError, McpResult};

#[derive(Debug, Clone, PartialEq, Eq, Hash)]
pub struct McpKey {
    pub tenant_id: String,   // "" = global
    pub server_name: String,
    pub version: String,
}

impl McpKey {
    #[must_use]
    pub fn new(tenant_id: impl Into<String>, server_name: impl Into<String>, version: impl Into<String>) -> Self {
        Self {
            tenant_id: tenant_id.into(),
            server_name: server_name.into(),
            version: version.into(),
        }
    }
    #[must_use]
    pub fn global(server_name: impl Into<String>, version: impl Into<String>) -> Self {
        Self::new("", server_name, version)
    }
}

#[derive(Default)]
pub struct McpSupervisor {
    clients: Mutex<HashMap<McpKey, Arc<dyn McpClient>>>,
}

impl std::fmt::Debug for McpSupervisor {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        f.debug_struct("McpSupervisor")
            .field("active", &self.clients.lock().keys().cloned().collect::<Vec<_>>())
            .finish()
    }
}

impl McpSupervisor {
    #[must_use]
    pub fn new() -> Self { Self::default() }

    pub async fn start(&self, key: McpKey, client: Arc<dyn McpClient>) -> McpResult<()> {
        // Replace silently: callers wanting "create new" semantics check get() first.
        if let Some(prev) = self.clients.lock().insert(key, client) {
            // Best-effort shutdown of the displaced client.
            let _ = prev.shutdown().await;
        }
        Ok(())
    }

    #[must_use]
    pub fn get(&self, key: &McpKey) -> Option<Arc<dyn McpClient>> {
        self.clients.lock().get(key).cloned()
    }

    pub async fn stop(&self, key: &McpKey) -> McpResult<()> {
        let removed = self.clients.lock().remove(key);
        match removed {
            Some(c) => c.shutdown().await.map_err(|e| {
                McpError::Transport(format!("shutdown {key:?}: {e}"))
            }),
            None => Ok(()),
        }
    }

    #[must_use]
    pub fn list_active(&self) -> Vec<McpKey> {
        self.clients.lock().keys().cloned().collect()
    }
}
```

## Risk & Fallback

| Risk | Mitigation |
|---|---|
| `rmcp 1.7` API doesn't match the field paths in Task 3 Step 5 (e.g. `RawContent` enum names) | Run `cargo doc -p rmcp --no-deps --open` and adjust. The *shape* of the code is correct; only field/method names may shift. Budget 30 min. |
| `cargo deny` flags a transitive of rmcp (e.g. a Unicode-3.0 or BSL-1.1 variant we don't have on the allowlist) | Add the SPDX to `deny.toml` `[licenses] allow` only if it's permissive. If it's a copyleft licence, file the dep as a workspace-level concern and fork/replace. |
| Fixture binary path resolution differs across `cargo test` invocation contexts (e.g. `cargo test --release`) | The `current_exe()` → walk-up logic handles both `target/debug` and `target/release`. If it breaks under nextest, switch to `CARGO_BIN_EXE_mock-mcp-server` env var (set by cargo for `[[bin]]`-tests in the same package). |
| MCP server child process not killed when `RunningService` is dropped, leaking `node` processes during test runs | rmcp documents that `RunningService` cancels on drop. If we still see leaked processes via `pgrep`, add an explicit `.cancel()` call inside `StdioMcpClient::Drop` — but this requires consuming `self`, so wrap `service` in `Option`. Defer to v0.5.3.1 if it bites. |
| `npx` is not available in CI | The e2e test is `#[ignore]`; CI runs without `--ignored`. The fixture-based test gives us the protocol coverage we need on every push. |
| Real MCP filesystem server tool names change (e.g., `read_file` → `read_text`) | The pinned subset is small (3 names) and well-established. If they change we update the test in a one-line commit; the failure surface is loud. |
