# v0.8.2 — chat-ui markdown + dark mode

**Status:** done.

## What shipped

Two chat-ui finishers from the v0.8.1 backlog:

1. **Markdown rendering** in assistant bubbles. The model's output is no
   longer flattened to plain text — headings, lists, tables, fenced code
   blocks, blockquotes and links render through `react-markdown` +
   `remark-gfm`. User and tool bubbles stay as raw text on purpose (users
   didn't author markdown; tool output is machine data).
2. **Dark mode** via a `[data-theme="dark"]` token fork on
   `document.documentElement`, plus a small three-way toggle in the
   sidebar footer: light · system · dark. The choice persists to
   `localStorage["xiaoguai-theme"]`; "system" follows
   `prefers-color-scheme` and recolours mid-session when the OS pref
   changes.

## Pieces

### `src/markdown.tsx`

Thin wrapper over `react-markdown` + `remark-gfm`. Raw HTML stays
escaped (the default) so the assistant can't inject `<script>` even if
jailbroken. Links open in a new tab with `rel="noreferrer noopener"`.

### `src/theme.ts`

`ThemeChoice = "light" | "dark" | "system"`. `useTheme()` hook owns
read/write of `localStorage` + dispatches the effective theme onto
`<html data-theme>`. `applyInitialTheme()` is called from `main.tsx`
*before* React mounts so the first paint avoids the
flash-of-light-on-dark.

### `src/ThemeToggle.tsx`

Segmented switch (☀ · ◐ · ☾) rendered into the sidebar via the new
`<SessionList>` children slot. Accessible: `role="radiogroup"` +
`aria-checked` per option.

### `styles.css`

* `:root` and `[data-theme='light']` share the same tokens — light is
  the default cascade target.
* New token: `--code-bg`, `--code-fg`, `--link`.
* `.md *` ruleset for markdown content: headings, lists, links, inline
  code, fenced code blocks, blockquotes, tables.
* `.theme-toggle` segmented-switch styling.
* `[data-theme='dark']` block forks every color token. Dark uses a
  near-black `#0b0c10` background, indigo accent, amber tool palette.
  `color-scheme: dark` flips form-control rendering too.

### Wire-through

* `App.tsx` mounts `<ThemeToggle />` via the new `<SessionList>` children
  slot.
* `ChatPage.tsx` only renders markdown for assistant bubbles that have
  finished streaming (the empty-streaming bubble stays in the
  3-dot-pulse state until the first delta lands).

## Acceptance

* `pnpm -F chat-ui build`: clean. CSS 8.33 kB (was 5.65 kB), JS 329 kB
  (was 170 kB). Increase is the `react-markdown` + `remark-gfm` bundle
  (~160 kB raw, ~50 kB gzip). Acceptable for the readability win;
  revisit with a lighter renderer or a code-split if the budget
  tightens.
* `pnpm -r typecheck`: clean across shared + chat-ui + admin-ui.
* Visual quality requires `pnpm -F chat-ui dev` + browser eyeballing.
