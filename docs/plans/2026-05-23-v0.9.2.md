# v0.9.2 — `xiaoguai-rag` crate

**Status:** done.

## What shipped

New workspace crate `xiaoguai-rag` — a thin adapter layer that gives
xiaoguai a RAG capability without rebuilding chunking / embedding /
retrieval / re-ranking from scratch. The user-locked principle ("wrap
a mature layer, don't build our own") translates to:

* **Production backend: R2R via HTTP** — SciPhi-AI's RAG service.
  Runs as a separate Docker process; we speak its REST API.
* **Dev / test backend: `InMemoryRagClient`** — substring-only, no
  embeddings, no chunking. Exists so eval cases and unit tests can
  run without R2R.
* **Both implement the same `RagClient` trait** — production wiring
  substitutes one for the other behind `Arc<dyn RagClient>`.

The crate also ships **`RagMcpAdapter`**, which wraps any
`RagClient` and implements `xiaoguai_mcp::McpClient`. So RAG appears
in the `Toolbox` exactly like any other MCP server — the agent loop
calls `rag_search` / `rag_ingest` / `rag_list_collections` /
`rag_delete_document` through the same dispatch path as a filesystem
or github MCP tool. Underscore separator (not dot) per the OpenAI
tool-name compatibility rule.

## The citation contract

Locked in this tag: every `SearchHit` MUST populate the full
citation envelope —

```rust
Citation {
    source_uri: String,       // file://, https://, or a custom scheme
    span: (u32, u32),         // 1-indexed line [start, end]
    score: f32,               // [0, 1]
    preview: String,          // chunk text (~200-400 chars typical)
    collection_id: String,
}
```

Backends that can't natively produce line spans MUST compute them at
ingest time from chunk offsets. **No silent unsourced text.** This
contract is what `ContentBlock::Citation` (v0.9.3) will deserialize
into, and what eval graders will assert against.

## Three chunking presets (no settings ocean)

`ChunkingPreset::{TextDefault, CodeAware, PdfHeavy}` → renders to the
JSON envelope R2R's `/v3/documents` expects. Advanced users can pass
raw `ingestion_config` to the backend if they really must; the agent
UI never exposes more than these three.

| Preset | Right for | Chunk size / overlap / strategy |
|---|---|---|
| `TextDefault` | Markdown notes, KB articles, prose | 512 / 64 / by_title |
| `CodeAware` | Source repositories | 256 / 32 / by_function |
| `PdfHeavy` | Academic / report PDFs | 1024 / 128 / by_page |

## Pieces

* `src/types.rs` — `Collection`, `Citation`, `SearchRequest/Result`,
  `IngestRequest/Result`. Single home for the wire shapes.
* `src/client.rs` — `RagClient` trait + `RagError`. Four methods:
  `list_collections`, `search`, `ingest`, `delete_document`.
* `src/memory.rs` — `InMemoryRagClient`. Substring search, line-level
  citations, idempotent ingest on `source_uri`.
* `src/r2r.rs` — `R2RClient`. HTTP via reqwest, R2R v3.x wire shapes.
  Citation lines taken from `metadata.line_start/end`; falls back to
  `(0, 0)` when R2R doesn't supply them (chunk-offset → line-number
  derivation lives in the *ingest* path which is a follow-up).
* `src/presets.rs` — `ChunkingPreset` enum + `IngestOptions`.
* `src/adapter.rs` — `RagMcpAdapter` exposing the four tools.

## Tests

`cargo test -p xiaoguai-rag`: **12 passed**.

* `memory::tests` (4): citation envelope completeness on search,
  idempotent ingest, NotFound on unknown collection, `min_score`
  filter.
* `presets::tests` (1): all three presets produce distinct R2R
  configs.
* `r2r::tests` (3): trailing-slash strip; `R2RChunkHit → Citation`
  envelope mapping (with and without `source_uri` metadata).
* `adapter::tests` (4): four tools listed; ingest→search round-trip;
  `list_collections` returns seeded entry; unknown tool surfaces
  `InvalidArgument`.

`tests/r2r_e2e.rs` is intentionally absent: a real R2R round-trip
needs Docker + a live R2R container, which we'd add later as
`#[ignore = "requires Docker"]` once we want to assert R2R version
compatibility automatically.

## Deferred (v0.9.2.x)

- **Wire `RagMcpAdapter` into `xiaoguai-core`'s `Toolbox` at
  startup.** Today the crate is built but nothing instantiates it.
  Pairs naturally with v0.9.4 (the marketplace) — RAG becomes the
  first preconfigured marketplace entry.
- **Chunk-offset → line-number derivation at ingest time.** Today
  the R2R adapter pass-through trusts R2R's `metadata.line_start/end`.
  When R2R doesn't populate it (which is the default for raw_text
  ingest), the citation is `(0, 0)`. Real-world line resolution is a
  follow-up.
- **Multi-collection search.** Today `SearchRequest` is single-
  collection. Cross-collection retrieval is an R2R `search_settings`
  knob away — we just need to widen the trait when we have a real
  use case.
- **Streaming ingest** (folder watcher). Belongs with v0.12.2 (the
  file watcher) — the crate trait surface accommodates it (ingest is
  per-document, idempotent on source_uri).
- **Real R2R e2e test** under `#[ignore = "requires Docker"]` once
  we want CI to verify wire compatibility.
