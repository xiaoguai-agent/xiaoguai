## Design notes

### Provider model (cross-cuts T1/T3)

```
LlmProvider {
    id: ProviderId,                  // ULID
    tenant_id: Option<TenantId>,     // None = system-wide
    name: String,                    // unique within (tenant_id, name)
    kind: ProviderKind,              // Ollama | OpenAiCompat
    endpoint: String,                // base URL
    models: Vec<String>,             // e.g. ["qwen2.5-coder", "deepseek-chat"]
    default_for_models: Vec<String>, // models for which this provider is preferred
    fallback_order: i32,             // ascending = tried first; sentinel -1 = never fallback
    api_key_env: Option<String>,     // resolve at backend-build time, not stored in DB
    created_at, updated_at
}

enum ProviderKind { Ollama, OpenAiCompat }
```

The secret stays in env (matches Section 14 of the design doc ŌĆö secrets via env, never DB). `api_key_env` names the env var; backend factory reads it.

### Resolution algorithm (T2)

```
resolve(req, ctx) ŌåÆ Vec<Arc<dyn LlmBackend>>:
    1. if req.provider explicit:        return [providers[req.provider]] + fallbacks
    2. if tenant default_for[req.model]: return that provider + fallbacks
    3. if system default_for[req.model]: return that provider + fallbacks
    4. else: return fallback chain ordered by fallback_order
    Filter out unhealthy breakers; if list empty ŌåÆ LlmError::NoProvider.
```

`Router::chat_stream` then walks this list. If the **first non-stream call** (the HTTP request to start the stream) returns `LlmError::Provider | Network`, the router moves to the next backend. Once a chunk has been forwarded to the caller, errors are propagated as-is (no mid-stream failover; the caller already has half a response).

### Token usage flow (T5)

`Router` wraps the inner stream in a `UsageRecordingStream` that accumulates an estimate (sums `delta.len()` as a coarse fallback) and, when the backend returns native usage fields (OpenAI does, Ollama 0.3.0+ does), prefers those. On `done: true` it sends one `UsageRecord` into a `tokio::sync::mpsc` channel; a background task batch-writes every 1 s or 100 records. Failure to enqueue (channel full) logs a warning and drops ŌĆö never blocks user-facing chat.

### Circuit breaker (T6)

Per-provider state:
```
enum BreakerState { Closed, Open { until: Instant }, HalfOpen }
```
Transition: Closed ŌåÆ(N failures within W) Open(until=now+30s) ŌåÆ(after `until`) HalfOpen ŌåÆ(success) Closed | (failure) Open. Configurable N=5, W=60s, cooldown=30s. State lives in `parking_lot::Mutex<HashMap<ProviderId, Breaker>>` inside `Router`.

## Risk & Fallback

| Risk | Mitigation |
|---|---|
| `eventsource-stream` cargo-deny licence (MIT/Apache; expect clean) | already in allowlist |
| OpenAI SSE format drift (vLLM occasionally omits `[DONE]`) | treat `Stream` end without `[DONE]` as `done: true` synthesised by us |
| RLS on `llm_providers` breaks because we connect as superuser in tests | tests use the same superuser path as other repos; behaviour matches `tenants` |
| Token counts unavailable for Ollama < 0.3.0 | record `None`; log a one-time warning per provider; do not fail |
| Breaker state lost on restart | acceptable for v0.5.2; persistence lands in v0.5.5 when API server gives us a place to keep cross-process state |
| Channel-based usage sink drops records under burst | back-pressure via channel; ADR-0009 (cost quota) gates request admission upstream in later milestones |
