# v0.8.1 — Chat UI design pass

Status: **Done** — 2026-05-23.

## Goal

The v0.8 styles.css was deliberately minimal — its docstring even said
"v0.8 focuses on shape, not polish. Real design system lands in v1.0."
v0.8.1 ships that design system: real typography, a small spacing
scale, refined colour tokens, subtle shadows, asymmetric bubble radii,
an animated streaming indicator.

## Caveat

The author of this slice (an LLM) cannot drive a browser. The CSS was
verified to compile + ship in the vite bundle, and the typecheck is
clean, but visual quality requires a human eye in front of `pnpm dev`.
If anything looks off, the design tokens at the top of `styles.css`
are the right place to start.

## What landed

### `frontend/chat-ui/src/styles.css`

Full rewrite. New design tokens at the top:
- Neutral-first colour palette (`--bg`, `--surface`, `--fg`, `--muted`,
  `--border`).
- 4-pt spacing scale (`--space-1` through `--space-7`).
- Three radii (`--radius-sm`, `-md`, `-lg`, `-xl`).
- Soft layered shadows (`--shadow-sm`, `--shadow-md`, `--shadow-focus`).
- Typography stack: Inter first (if available), then system fonts.

Layout polish:
- Conversation column capped at 720px and centered on wide screens so
  the eye doesn't sweep the full monitor.
- Composer line aligns with the conversation column.
- Mobile breakpoint at 720px stacks sidebar above messages.

Bubble polish:
- Asymmetric border-radius creates a subtle "tail" toward the right
  for user bubbles and the left for assistant bubbles.
- Tool bubbles render in a monospace stack (JetBrains Mono / IBM Plex
  Mono / system mono fallback) so machine output reads as such.
- Refined accent + accent-soft colours for active session highlight.

### `frontend/chat-ui/src/ChatPage.tsx`

`Bubble` component renders three animated dots (`.streaming-dots`)
when a bubble is empty + streaming, instead of an inert ellipsis.
Pure presentational change; no logic touched.

### Sidebar

- Section heading uses uppercase + letter-spacing as a label rather
  than a heavy `<h2>`.
- "New session" button styled as a quiet outlined control instead of
  loud blue.
- Active session uses `--accent-soft` (light-tinted background)
  rather than a separate `color` change so the contrast stays calm.

### Streaming indicator

- New `.streaming-dots` selector with a three-step `@keyframes pulse`.
- Stagger via `animation-delay: 0s/0.15s/0.3s`.
- 1.2s loop, ease-in-out.

## Deferred

- **Dark mode.** The token structure supports forking into
  `[data-theme="dark"]` but no dark palette is shipped.
- **Code-block syntax highlighting.** Tool bubbles use monospace but
  there's no actual highlighter (Prism / Shiki). Adding it means a
  bundle-size tradeoff.
- **Markdown rendering in assistant bubbles.** Current `white-space:
  pre-wrap` is functional but headers / lists / inline code don't
  render. A markdown-it / react-markdown integration is its own
  slice.
- **Avatars** next to each message. The current layout reads cleanly
  without them; revisit when there are multi-user / multi-agent chats.
- **Settings / preferences pane** (theme toggle, font size, etc).
- **Accessibility audit.** Tab order + ARIA labels are minimal; a
  proper pass is its own slice.
